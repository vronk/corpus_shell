Parameter base_url
Documentation

Description

A URL that is used by this style sheet to build links and fetch additional data

Corresponds to //sru:baseUrl. If this is an empty string links used in this style sheet point back to the URL that executed the style sheet. If this is not an SRU aware endpoint strange warnings and behaviors can be expected.

Examples:

  • http://corpus3.aac.ac.at/switch
  • http://clarin.aac.ac.at/cr/lrp/fcs
  • http://localhost/corpus_shell/modules/fcs-aggregator/switch.php
  • The URL has to be reachable by the XSLT processor so sth. like this is possible: http://localhost:8686/exist/rest/db/cr/cr.xql

Defaults to an empty xs:string.

Parameter scripts_url
Documentation

Description

A URL for locating JavaScript scripts and CSS style sheets

Note: If this parameter is supplied it has to end in a path separator (/)!

Note: These paths are meant for the user's browser so localhost is usually not an option!

If this is an empty string the location of the files are interpreted relativ to the URL that executed the style sheet respectively the document generated by the style sheet.

Usually passed by the script switch.php.

Examples:

Parameter format
Documentation

Description

Requested format of the result

One of htmlpage, htmljspage or htmlsimple or something completely different. Most importantly this chooses between four different frameworks for the page with more or less building blocks. Unrecognizeable strings generate only the least possible amount of HTML.

Possible intended future use: Mix and match parts needed by the application:

One a combination of

  • "html" to actually generate an HTML framework as opposed to an AJAX HTML fragment + one of
    • "page"
    • "js" generate an HTML page that utilises JavaScript
    • "simple" generate a simple HTML page
  • "list" represent the SRU items as a list
  • "detail" add detail information

Examples would be: htmlsimpledetail or htmljslist

Defaults to 'htmlpagelist'.
