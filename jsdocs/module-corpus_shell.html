<inner> doOnDocumentReady()

  1. Bind events to DOM elements that may not exist in the current DOM tree using jQuery. These DOM elements are generated by some search request results so further functionality can be provided.
    • For every link in a .data-view .full class container in a .searchresults the default behavior is replaced by the module:corpus_shell~PanelManager#OpenSubPanel method. This method is used to open a full text sub panel.
    • For every link in a .data-view .full class container in a .searchresults the default behavior is replaced by the module:corpus_shell~PanelManager#OpenSubPanel method. This method is used to open an image sub panel which shows the facsimile.
    • For every link of class .value-caller in a .searchresults for a scan of Schnitzler Tagebücher the default behavior is replaced by the module:corpus_shell~PanelManager#OpenSubPanel method. This method experimental and not working as intended right now.
      TODO: Intended behavior: On a click in the Scan panel open a Search panel using the search provided in the Scan panel's link.
    • For every link in a .navigation class container in a .searchresults the default behavior is replaced by the module:corpus_shell~PanelManager#OpenSubPanel method. This method is used to enable page navigation in full text views.
  2. Tries to get a user id.
  3. Initiates asynchrounous loading of the profiles and the index data.
  4. Installs a handler for the module:corpus_shell~PanelManager#event:onUpdated event of module:corpus_shell~PanelController.
  5. Displays a URL which contains the current user ID in a DOM element designated by #userid
  6. Generates a drop down list with search targets using module:corpus_shell~GenerateSearchCombo into the DOM element designated by #searchbuttons and preselects the first item
  7. Generates a drop down list with profiles using module:corpus_shell~GenerateProfileCombo right after the DOM element designated by #profiledel and preselects the first item

<inner> GetUserData(userId)

Tries to retrieve profile settings saved on the server.
If contacting the the script fails the browsers localstore is used.

Retrieval is done asynchronously so this function returns immediately and the actual result may be be valid later.

  2. Passes the data to module:corpus_shell~ProfileController as module:corpus_shell~ProfileManager#Profiles
